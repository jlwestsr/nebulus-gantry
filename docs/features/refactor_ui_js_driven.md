# Feature: Refactor to JS-Driven UI with REST API

## 1. Overview
**Branch**: `feat/refactor-ui-js-rest`

This feature refactors the application's frontend architecture to be fully JavaScript-driven, interacting exclusively with a backend REST API. It addresses the technical debt of inline styles and mixed concerns by strictly separating HTML structure, CSS styling, and JavaScript logic. The goal is a cleaner, more maintainable, and standard-compliant codebase.

## 2. Requirements
List specific, testable requirements:
- [ ] **Strict Separation**: HTML files must contain only structure (no `style="..."`), CSS files only styling, and JS only logic.
- [ ] **No Inline Styles**: All inline styles in Python templates (`ui/`) and JavaScript injections (`public/`) must be replaced with CSS classes.
- [ ] **REST API Consistency**: All frontend data fetching and state mutation must occur via `fetch` calls to the existing REST API endpoints.
- [ ] **Semantic HTML**: HTML generated by Python functions must use semantic tags and proper `id` / `class` attributes for targeting.

## 3. Technical Implementation
- **Modules**:
    - `ui/pages.py`: Remove inline styles, return clean HTML skeletons.
    - `ui/model_builder.py`: Extract CSS to external file, clean HTML.
    - `public/style.css`: Add general utility classes and extracted styles.
    - `public/workspace.css`: Add styles extracted from `model_builder.py` and `workspace_page`.
    - `public/notes.js`: Refactor to use class toggling instead of direct style manipulation.
    - `public/workspace.js`: Refactor to use class toggling and API-driven updates.
- **Dependencies**: None.
- **Data**: None.
